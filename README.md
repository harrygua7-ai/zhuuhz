# 热点跟稿官

发现赛道热点，用你的口吻生成发布就绪稿。

热点跟稿官是一个面向个人 IP、独立创作者和一人公司的 Codex/BotLearn Skill。它会接收用户提供的热点，或在支持搜索的环境中扫描最近 0-48 小时相关热点，先做事实核查和信源分级，再判断这个话题值不值得跟，最后按用户的语言风格生成可人工确认发布的稿子。

## What It Does

- 扫描或接收赛道热点、新闻、链接、截图文字、RSS/监控器候选结果
- 对事实做 L1/L2/L3 信源分级，标注已验证、待确认和存在争议的信息
- 判断热点相关度、事实可信度、IP 适配度、原创改编空间和抄袭/撞稿风险
- 生成两类发布就绪稿：客观蹭热度稿、个人 IP 观点稿
- 输出标题、正文、首图/封面文案、评论区引导、事实来源和质检评分
- 在无搜索能力时降级运行，明确提示事实无法独立验证

## What It Does Not Do

- 不自动登录微博、小红书、抖音、视频号、X、LinkedIn 等平台
- 不模拟人工点击，不自动发布，不群控，不批量评论
- 不承诺全网实时无遗漏监控
- 不把未验证信息写成确定事实
- 不生成搬运、洗稿或逐字改写他人爆款内容

## Quick Start

Skill 目录：

```text
ip-trend-draft-copilot/
  SKILL.md
  agents/openai.yaml
  references/output-templates.md
```

默认推荐入口：

```text
A. 立即扫描最近 0-48 小时相关热点并生成稿子（推荐）
```

示例使用：

```text
Use $ip-trend-draft-copilot to scan recent 0-48 hour AI product trends and draft a publish-ready post in my voice.

我是 AI 产品独立开发者，写给创业者和产品经理。
风格：短句、直接、有一点毒舌。
红线：不谈政治，不攻击具体个人，不做无来源判断。
```

如果当前环境不支持联网/搜索，可以改用：

```text
我手上已有热点，帮我判断值不值得写。

热点材料：
...

我想要：个人 IP 稿
```

## Safety Boundary

热点跟稿官只生成发布就绪内容和审核材料。用户需要自己确认事实、语气和平台规则后手动发布。对于金融、政策、法律、医疗、安全和声誉风险类内容，Skill 会更严格地标注不确定性，并优先建议谨慎发布或不发布。
